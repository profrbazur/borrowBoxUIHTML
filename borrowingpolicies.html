Eligibility Toggles

This section controls whether users are allowed to borrow based on overdue items, unpaid fines, reservation policies, and other operational restrictions.

Block Overdue Users

Prevent new borrowing if a user has overdue items.

Require Unpaid Fine Clearance

Users must settle unpaid fines before requesting new items.

Allow Reservation Transfers

Permit staff to reassign a reservation to another eligible user.

Allow Multiple Active Requests

Allow users to submit more than one active request at a time.

Allow Cross-Department Borrowing

Permit borrowers to request items owned by another department.

Borrowing Control Settings

Fine-tune request concurrency, reservation auto-cancel behavior, and borrower account restrictions.

Auto-Cancel Unclaimed Reservations

Automatically release reserved items once the hold duration expires.

Restrict Inactive Accounts from Borrowing

Inactive accounts will not appear in approval and release flows.

Require Verified Accounts Only

Only validated users can proceed with requests and reservations.

Policy Summary

Default circulation rule applies when no specific user type rule exists.
Category overrides take precedence over standard borrow rules.
Students can borrow up to 1 item for 3 days.
Faculty can borrow up to 2 items for 90 days.
Overdue users are blocked from borrowing.
Unpaid fines must be cleared before new borrowing.
First reminder is sent 24 hours before due date.
Borrowers are flagged as problem users after 30 overdue days.
